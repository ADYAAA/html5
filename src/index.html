HTML5 ★ Boilerplate

{% blocktrans %}A rock-solid default for HTML5 awesome.{% endblocktrans %}

{% blocktrans %}HTML5 Boilerplate is the professional badass's base HTML/CSS/JS template for a fast, robust and future-proof site.{% endblocktrans %}

{% blocktrans %}After more than two years in iterative development, you get the best of the best practices baked in: cross-browser normalization, performance optimizations, even optional features like cross-domain Ajax and Flash. A starter apache .htaccess config file hooks you the eff up with caching rules and preps your site to serve HTML5 video, use @font-face, and get your gzip zipple on.{% endblocktrans %}

{% blocktrans %}Boilerplate is not a framework, nor does it prescribe any philosophy of development, it's just got some tricks to get your project off the ground quickly and right-footed.{% endblocktrans %}

INDEX.HTML

  • Comment about the protocol-relative URL in use.
  • Standardize comment syntax and grammar.
  • The order of <meta> tags, <title>, and charset has been documented more extensively now you are safe to use the boilerplate's order.
  • Scrunch up the analytics snippet some more. It doesn’t need to be so readable IMO
  • Going back to non-$.getScript'd DD_belatedPNG, due to some messy race conditions.
  • Added role to div#main for elemental ARIA.
  • Clean up new async DD_belated call.
  • Fixed modernizr path issue.
  • Implement async load for DD_belated

STYLE.CSS

  • Add .focusable helper class that extends .visuallyhidden to allow the element to be focusable when navigated to via the keyboard.
  • Removed outline: 0 inline with Eric Meyer's new reset.
  • Removed the resetting of anchor links.
  • Applying list-style-image:none to nav ul in all browsers (necessary for IE7-).
  • Moved helper classes after primary styles to make sure they override author styles.
  • Updated reset to latest updates to Eric Meyer's reset css.
  • Removed camelcase from the classname visuallyhidden to be consistent with other classname formats.
  • Updated visuallyHidden rule to make sure it overrides all other declarations. Thanks Jonathan Neal.
  • Removed reset on the image in a table cell as it looks terrible with multiple lines. Browser defaults to baseline alignment for images which works better than top alignment.
  • Increased margin-left on ol, to allow for 2-digit list numbers. Thanks MichaelvanLaar.
  • Added a reset on filters for print styles.
  • Updated sub/sup css to make them not be impacted by line-height.
  • Cleaning up print link styles - Do not print hash-links, do not print href-values for javascript links.
  • Black color prints faster.
  • Added input type=image based on Ivannikolic's suggestion
  • Removed IE9 conditional on html file.

WEB SERVER CONFIGS

  • Caching of .htc files.
  • Use the .appcache extension instead of .manifest, as per http://html5.org/r/5812.
  • Header directive appeared without IfModule mod_headers.c causing Internal Server Error when mod_headers was not available.
  • Htaccess: block access to .git and .svn folders.
  • Eradicating chrome console warning on WOFF font downloads.
  • ServerTokens Prod added to htaccess. thx alrra.
  • Changing mimetype of .ico files to "image/x-icon".
  • Moved all server configurations (except apache's .htaccess) over to a new repo
  • Force deflate for mangled accept-encoding headers
  • .htaccess: some additional features and more structure for easier readability
  • Added Flash video mime types to IIS server
  • Clean up cache expires directives in nginx.conf
  • Ensure that .htaccess works even if mod_headers is not installed
  • Changed nginx.conf to include SVG and font formats for gzipping
  • Nginx already gzips text/html. we can avoid a warning.
  • Nginx config expires bugfix.
  • Added plugin/extension mime types for safari, chrome, firefox
  • Updated .htaccess and mime.types for ogg formats.
  • Added nginx mime.types config file with the MIME types additions made in .htaccess file.
  • Initial commit of a lighttpd config file intended to be equivalent in functionality to the apache and nginx config files
  • Fixed the filter module to reflect only html, txt, css, js, json, xml, htc
  • Fixed regression where EOT fonts had been excluded from DEFLATE compression
  • Removed rss/atom & MS formats not necessary for generic use
  • Updated .htaccess regex for filter_module thanks to unwiredbrain
  • Add note and link regarding best practices for Apache config
  • Use mod_filter for compression, with fallback to AddOutputFilterByType directive for legacy versions
  • Append instead of setting Cache-Control:public header so it doesn't get clobbered by ExpiresByType

LIBRARIES

  • Updated modernizr-1.6.min.js to modernizr-1.7.min.js
  • Update to jQuery 1.5.1 now that ajax.googleapis.com finally hosts it as well.

OTHERS

  • Added humans.txt.
  • note: you can add page handlers for 500, 403 errors easily.
  • Removed YUI profiling. you probably weren't using it anyway.
  • Removed qunit's unit tests. no need to ship with them, really.
  • Add ignores for most common editors so that swap files don't show up as untracked files.
